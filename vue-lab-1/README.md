# Лабораторна робота №1

# Тема
Розгортання простого Vue.js додатку з використанням Docker.

## Мета
Оволодіти основними кроками створення, налаштування та розгортання Vue.js додатку, а також ознайомитися з використанням контейнерізації за допомогою Docker для легкого розповсюдження та збереження середовища розробки.

## Теоретичні відомості

Vue.js - це прогресивний фреймворк для створення веб-інтерфейсів, який спрощує розробку веб-додатків та дозволяє ефективно керувати їхнім станом та компонентами. Для роботи з Vue.js зазвичай використовують Vue CLI - інструмент командного рядка для швидкого створення та налаштування проектів.

### Компоненти
Vue.js базується на компонентах - це відокремлені блоки коду, які можна перевикористовувати та комбінувати для створення складних інтерфейсів.
### Директиви
Директиви - це спеціальні атрибути, які вказують Vue, як повинен маніпулювати DOM-елементами. Наприклад, `v-for`, `v-if`, `v-bind`.
### Власні події та властивості
Vue дозволяє створювати власні події для спілкування між компонентами, а також використовувати властивості для передачі даних вниз по ієрархії компонентів.

## Завдання

### Крок 1: Ознайомлення з теоретичним матеріалом
Перш ніж розпочати роботу, ретельно ознайомтеся з документацією Vue.js та зрозумійте основи роботи з цим фреймворком. Ви можете знайти документацію на офіційному сайті Vue.js.

### Крок 2: Встановлення Node.js та npm
Переконайтеся, що на вашому комп'ютері встановлена остання стабільна версія Node.js та менеджера залежностей npm. Ви можете завантажити їх з офіційного сайту Node.js і встановити за інструкцією на сайті.

### Крок 3: Створення елементарного додатку за допомогою vue-cli
1. Встановіть глобально утиліту vue-cli, якщо вона ще не встановлена, за допомогою команди npm:

```bash
npm install -g @vue/cli
```

2. Створіть новий Vue.js проект за допомогою vue-cli, виконавши команду:

```bash
npm init vite@latest <ваше-прізвище-англійською> -- --template vue
```

Слідуйте інструкціям на екрані для налаштування проекту.

### Крок 4: Тестування роботи додатку

Запустіть ваш новий додаток за допомогою команди:

```bash
cd <ваше-прізвище-англійською>
npm install
npm run dev
```

Відкрийте браузер і перейдіть за адресою `http://localhost:8080` (або інша адреса, яка виведена в терміналі). Ви повинні бачити ваш елементарний Vue.js додаток у роботі.

### Крок 6: Редагування компоненту

Знайдіть компонент HelloWorld.vue та додайте інформацїю про себе (Спеціальність, курс, група, ПІБ) у шаблоні компонента.

### Крок 7: Створення Dockerfile та .dockerignore

Створіть файл `Dockerfile` у кореневій директорії вашого проекту з наступним вмістом:

```Dockerfile
FROM node:16 as build-stage
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build
FROM nginx:stable-alpine as production-stage
COPY --from=build-stage /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

Створіть файл `.dockerignore` для виключення непотрібних файлів та директорій з образу Docker:

```
node_modules
```

Додайте додаткові команти в секції scripts в файлі package.json

```
...
"docker-run": "docker build --tag vue-lab-1:latest . && docker run --rm -p 8080:80 --name vue-lab-1-instance vue-lab-1:latest",
"docker-stop": "docker stop vue-lab-1-instance"
...
```
Виконайте команду docker-run та відкрийте браузер на сторінці http://localhost:8080/
Зупиніть контейнер виконавши docker-stop.

### Крок 6: Розміщення проекту на GitHub або Bitbucket

Зареєструйтесь на GitHub або Bitbucket, якщо ви ще цього не зробили. Після цього створіть новий репозиторій з іменем "mjsf-mag" на одному з цих сервісів.

### Крок 7: Завантаження проекту в репозиторій

Використовуйте команди Git для завантаження вашого проекту до репозиторію. Вам потрібно буде виконати наступні команди:

```bash
git init
git add .
git commit -m "Перший коміт: додано Vue.js додаток"
git branch -M main
git remote add origin <посилання-на-ваш-репозиторій>
git push -u origin main
```

### Крок 8: Відправлення посилання на репозиторій
Посилання на ваш репозиторій "mjsf-mag" на GitHub або Bitbucket ви можете відправити як звіт про виконану роботу.
