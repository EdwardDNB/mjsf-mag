# Лабораторна робота: Компонент для завантаження файлів з Drag&Drop та прев’ю документів

## Мета

Навчитися створювати **Vue 3 компонент для завантаження файлів** з підтримкою drag&drop.  
Компонент повинен показувати **прев’ю завантажених файлів** та заздалегідь підготовлені **іконки документів**, що передаються через пропси.

## Пропси компонента

| Пропс            | Тип                  | Опис                                           |
|-----------------|--------------------|-----------------------------------------------|
| `modelValue`     | `Array<File>`       | Масив обраних файлів (v-model)               |
| `documents`      | `Array<{ type: string; icon: string; name: string }>` | Масив заготовлених документів для прев’ю |
| `maxFiles`       | `number`           | Максимальна кількість файлів                 |

---

## Події

| Подія               | Тип                | Опис                                      |
|-------------------|------------------|------------------------------------------|
| `update:modelValue` | `Array<File>`     | Викликається при додаванні/видаленні файлів |

---

## Приклад використання

```vue
<script setup lang="ts">
import { ref } from 'vue'
const uploadedFiles = ref<File[]>([])
const availableDocuments = [
  { type: 'pdf', icon: '/icons/pdf.svg', name: 'Contract.pdf' },
  { type: 'docx', icon: '/icons/docx.svg', name: 'Report.docx' },
  { type: 'xlsx', icon: '/icons/xlsx.svg', name: 'Data.xlsx' }
]
</script>

<template>
  <DocumentUploader
    v-model="uploadedFiles"
    :documents="availableDocuments"
    :maxFiles="5"
  />
</template>
```

## Завдання

Реалізувати компонент `<DocumentUploader>` з наступними можливостями (можна використовувати приклад з лекції):

### 1. Drag&Drop та вибір через клік
- Додавати файли через drag&drop або клік на зону завантаження.

### 2. Прев’ю завантажених файлів
- Для кожного завантаженого файлу створювати прев’ю з використанням `URL.createObjectURL(file)`.

### 3. Прев’ю заздалегідь підготовлених документів
- Відображати документи з пропса `documents` з іконками та назвами.

### 4. Видалення документів
- Додати кнопку видалення для кожного прев’ю, як завантаженого, так і передпідготовленого.

### 5. Обмеження кількості файлів
- Забезпечити обмеження по `maxFiles`.

### 6. Оформлення стилями
- Використовувати CSS/SCSS для гарного відображення сітки документів та прев’ю.

### 7. Оформити компонент як плагін
- Створити `index.ts` з функцією `install(app: App)`.  
- Реєструвати компонент глобально через плагін.  
- Підготувати документацію у `README.md` для npm.  
- Забезпечити підтримку TypeScript через `.d.ts` файли.

### 8. Публікація пакета на npm
- Підготувати пакет для публікації.  
- Створити `package.json` з усіма необхідними полями (`name`, `version`, `main`, `types`).  
- Виконати `npm publish` для розповсюдження пакета.
