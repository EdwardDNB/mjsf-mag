# Лабораторна робота №1: Розгортання Vue.js додатку з використанням Docker

## Мета
Оволодіти основними кроками створення, налаштування та розгортання Vue.js додатку, а також ознайомитися з використанням контейнеризації за допомогою Docker для легкого розповсюдження та збереження середовища розробки.

## Теоретичні відомості

### Контейнеризація
**Контейнеризація** — це сучасний підхід до розробки програмного забезпечення, який забезпечує ефективний, зручний і безпечний спосіб створення, доставки та запуску застосунків.

Основні переваги контейнеризації:
- **одноманітність і незалежність** — застосунок працює однаково в будь-якому середовищі;
- **кросплатформеність** — можливість запуску на різних операційних системах і інфраструктурах;
- **ізоляція та безпека** — контейнери відокремлені один від одного та від хост-системи;
- **ефективне використання ресурсів** — контейнери «легші» за віртуальні машини;
- **швидке розгортання та масштабування** — легко запускати нові екземпляри й керувати навантаженням.

Одним із найпопулярніших інструментів для контейнеризації є **Docker**.

**Docker** — це програмна платформа для швидкої розробки, тестування й розгортання застосунків. Вона пакує програмне забезпечення у стандартизовані блоки, що називаються **контейнерами**. Кожен контейнер містить усе необхідне для роботи застосунку: бібліотеки, системні інструменти, код і середовище виконання. Завдяки цьому можна гарантувати стабільну роботу програмного забезпечення незалежно від середовища, у якому воно запускається.

**Docker Compose** — це інструмент для керування багатоконтейнерними застосунками в Docker. Він дозволяє визначати, налаштовувати та запускати кілька взаємопов’язаних сервісів за допомогою єдиного конфігураційного файлу у форматі **YAML**. Це значно спрощує роботу з комплексними системами, де застосунок складається з декількох компонентів (наприклад, веб-сервер, база даних і кеш).

### Менеджер пакетів

**Node.js** — це середовище виконання JavaScript, побудоване на рушії **V8** від Google Chrome. Воно дозволяє запускати JavaScript не лише у браузері, а й на сервері. 
Завдяки цьому розробники можуть використовувати одну мову програмування — JavaScript — як для клієнтської, так і для серверної частини застосунків.

**NPM (Node Package Manager)** — це стандартний менеджер пакетів для Node.js, який дозволяє легко встановлювати, оновлювати й керувати сторонніми бібліотеками та фреймворками.

Він надає низку команд для керування пакетами в проєкті. Розглянемо приклади основних із них:
 - `npm install package-name`_ – встановлення пакета. Ця команда встановить пакет з ім’ям package-name і додасть його у ваш файл package.json у секцію залежностей.
 - `npm update package-name` – оновлення пакета. Команда оновить встановлені пакети до їхніх останніх версій, з урахуванням вашої версійної політики у файлі package.json.
 - `npm remove package-name` – видалення пакета. Команда прибере пакет package-name з вашого проєкту і відповідним чином оновить файл package.json.
За умови правильного використання даних команд, вони допомагають підтримувати проєкт в актуальному стані та забезпечують легкість управління його розвитком.

Однією з переваг **npm** є можливість визначення користувацьких скриптів у секції scripts файлу package.json. Ці скрипти можуть виконувати різні завдання, як-от збірка, тестування, запуск сервера та багато іншого.

Приклади scripts у package.json:
```json
"scripts": {
  "start": "node server.js",
  "build": "webpack --config webpack.config.js",
  "test": "mocha tests/**/*.js"
}
```
У репозиторії NPM доступно понад 1,3 мільйона пакетів. Ось деякі з найбільш популярних:

- **express** – фреймворк для створення веб-додатків на Node.js. Застосування: розробка серверної частини веб-додатків, обробка маршрутів і HTTP-запитів.
- **react** – бібліотека для створення користувацьких інтерфейсів. Застосування: розробка компонентів користувацького інтерфейсу в односторінкових додатках.
- **angular** – фреймворк для створення односторінкових веб-додатків. Застосування: розробка структурованих і масштабованих веб-додатків.
- **vue** – прогресивний фреймворк для створення односторінкових веб-додатків. Застосування: розробка гнучких і легковажних користувацьких інтерфейсів.
- **axios** – HTTP-клієнт для виконання запитів до зовнішніх API. Застосування: взаємодія із зовнішніми серверами, надсилання та отримання даних.
- **lodash** – бібліотека для роботи з колекціями даних, такими як масиви та об’єкти. Застосування: маніпуляції даними, опрацювання колекцій і керування функціями.

### Vue.js

**Vue.js** — це прогресивний JavaScript-фреймворк, який використовується для створення сучасних користувацьких інтерфейсів та односторінкових застосунків. 
Його головною перевагою вважають простоту: синтаксис Vue інтуїтивний, а офіційна документація є зрозумілою навіть для початківців. 
Завдяки цьому вхід у технологію стає легшим, ніж у багатьох інших фреймворках.
Основою Vue є концепція реактивності — будь-які зміни у даних миттєво відображаються в інтерфейсі без необхідності прямого втручання у DOM. 
Це досягається завдяки механізму двостороннього зв’язку, який дозволяє синхронізувати дані між моделлю та відображенням. 
Водночас Vue використовує віртуальний DOM, що забезпечує ефективне оновлення інтерфейсу і зменшує кількість непотрібних операцій з реальним DOM.

Архітектура Vue побудована на компонентному підході. Будь-який інтерфейс можна розділити на окремі багаторазові блоки, які легко підтримувати й комбінувати. 
Цей підхід робить фреймворк придатним як для невеликих інтеграцій у вже готові сторінки, так і для побудови масштабних систем. 
Крім того, Vue має багату екосистему офіційних і сторонніх інструментів. 
Для організації маршрутизації існує бібліотека Vue Router, для керування станом застосунків використовується Pinia (раніше Vuex), а для серверного рендерингу та статичної генерації застосунків створено Nuxt.js.

Сучасна версія Vue, яка розроблялася з урахуванням підтримки TypeScript, забезпечує зручність роботи для тих команд, що прагнуть використовувати типізацію у своїх проєктах. 
Активна спільнота й велика кількість готових плагінів та UI-бібліотек, таких як Vuetify, Element Plus чи Naive UI, дозволяють значно прискорювати процес розробки.
Vue.js вирізняється поєднанням простоти та потужності. Він легший у вивченні за Angular і водночас пропонує більш структурований підхід порівняно з React. 
Саме ця збалансованість робить його популярним вибором для створення як невеликих застосунків, так і складних масштабованих проєктів.


## Завдання

1. Ознайомтеся з документацією Vue.js, щоб зрозуміти основи роботи з фреймворком. Документацію можна знайти на [офіційному сайті Vue.js](https: //vuejs.org/).
2. Встановіть **Node.js** та **npm** та **Docker** у вашому локальному середовищі.
3. Створіть новий проект Vue.js за допомогою Vite
   ```bash
   npm create vite@latest <ваше-прізвище-англійською> -- --template vue
   ```
4. Запустіть проект та переконайтеся, що він працює коректно.
   ```bash
   cd <ваше-прізвище-англійською>
   npm install
   npm run dev
   ```
5. Знайдіть компонент HelloWorld.vue та додайте інформацію про себе (Спеціальність, курс, група, ПІБ).
6. Створіть `Dockerfile` для збірки та запуску Vue.js додатку.

    ```DOCKERFILE
   # Build stage: використовуємо офіційний образ Node.js для збірки Vue додатку
   FROM node:latest as build-stage
   # Встановлюємо робочу директорію всередині контейнера
   WORKDIR /app
   # Копіюємо тільки package.json та package-lock.json для кешування npm install
   COPY package*.json ./
   # Встановлюємо залежності
   RUN npm install
   # Копіюємо весь код проекту у контейнер
   COPY . .
   # Збираємо production-версію Vue додатку
   RUN npm run build
   
   # Production stage: використовуємо легкий Nginx для сервінгу статичних файлів
   FROM nginx:stable-alpine as production-stage
   # Копіюємо збірку з build-stage у Nginx
   COPY --from=build-stage /app/dist /usr/share/nginx/html
   # Відкриваємо порт 80 для доступу до веб-додатку
   EXPOSE 80
   # Команда для запуску Nginx у foreground (обов'язково для Docker)
   CMD ["nginx", "-g", "daemon off;"]
   ```
7. Додайте `.dockerignore` для виключення непотрібних файлів:

   ```dockerignore
   node_modules
   dist
   .git
   .env
   *.log
   ```
8. Створіть файл `docker-compose.yml` для спрощення керування контейнером:

   ```yaml
   services:
     vue-app:
       build: .
       ports:
         - "8080:80"
       container_name: vue-lab-1-instance
   ``` 
9. Збережіть `Dockerfile` та `.dockerignore` у кореневій директорії проэкту.
10. Додайте до package.json наступні команди:

   ```json
      "scripts": {
        "docker-run": "docker-compose up -d --build",
        "docker-stop": "docker-compose down",
      }
   ```
11. Протестуйте роботу додатка та зупиніть його за допомогою команд:

   ```bash
   npm run docker-run
   ```
   Перевірте, що додаток доступний за адресою `http://localhost:8080`.
   Зупиніть додаток:

   ```bash
   npm run docker-stop
   ```
12. Розмістіть ваш проект в директорії `lab1` публічномого репозиторію GitHub/GitLab/Bitbucket з назвою `<vendor>/mjsf-mag` та додайте посилання на репозиторій у звіт виконавши наступні команди:

   ```bash
   git init
   git add .
   git commit -m "Vue.js lab 1"
   git branch -M main
   git remote add origin <посилання-на-репозиторій>
   git push -u origin main
   ```
